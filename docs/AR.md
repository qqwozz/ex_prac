
# Exam Trainer — Архитектура приложения

Подробная схема того, как всё работает внутри. Для понимания потоков данных и взаимодействия компонентов.

## Потоки данных

### 1. Получение задания

```
Ученик → Фронтенд → Python → Go → Supabase → Go → Python → Фронтенд → Ученик

1. Ученик нажимает «Следующее задание»
2. Фронтенд отправляет GET /api/v1/tasks?subject=math&topic=geometry
3. Python проксирует запрос в Go (8080)
4. Go делает SELECT * FROM tasks WHERE ...
5. Supabase возвращает задание
6. Go возвращает JSON
7. Python проксирует обратно
8. Фронтенд отображает задание
```

### 2. Проверка ответа

```
Ученик → Фронтенд → Python → Go → Supabase → Go → Python → Фронтенд → Ученик

1. Ученик вводит ответ и нажимает «Ответить»
2. Фронтенд отправляет POST /api/v1/check { task_id, answer }
3. Python проксирует в Go
4. Go:
   a. Достаёт правильный ответ из Supabase
   b. Сравнивает с ответом ученика (с учётом типа задания)
   c. Сохраняет попытку в attempts
   d. Возвращает { correct, correct_answer, explanation }
5. Python проксирует обратно
6. Фронтенд показывает результат
```

### 3. AI-подсказка

```
Ученик → Фронтенд → Python → DeepSeek → Python → Фронтенд → Ученик

1. Ученик нажимает «Подсказка»
2. Фронтенд отправляет POST /ai/v1/hint { task_id, user_answer }
3. Python:
   a. Проверяет лимит (hints_used_today < 10)
   b. Формирует промт для DeepSeek
   c. Вызывает DeepSeek API
   d. Если DeepSeek недоступен → OpenAI
   e. Возвращает { hint, provider }
4. Фронтенд показывает подсказку
```

### 4. Адаптивное обучение

```
Ученик → Фронтенд → Python → Go → Supabase → Go (адаптив) → Go → Python → Фронтенд

1. Ученик выбирает режим (улучшить/подтянуть)
2. Фронтенд отправляет GET /api/v1/tasks?mode=improve
3. Go:
   a. Считает % правильных по модулям из attempts
   b. Сортирует модули
   c. Выбирает подходящий модуль и уровень сложности
   d. Возвращает задание
4. Фронтенд показывает
```

### 5. Проверка подписки

```
Ученик → Фронтенд → Python → Go → Supabase → Go → Python → Фронтенд

1. Пользователь открывает сервис
2. Python запрашивает Go: GET /api/v1/user/{id}
3. Go проверяет subscription_type в Supabase
4. Если basic → показываем баннер «Оформите Плюс»
5. Если plus → полный доступ
6. Проверка кэшируется на время сессии (не чаще раза в 5 минут)
```

### 6. Добавление задания (препод)

```
Препод → Telegram-бот → Go → Supabase

1. Препод отправляет задание в Telegram-бот
2. Бот парсит и форматирует
3. Отправляет POST /api/v1/tasks в Go
4. Go сохраняет в Supabase
```

### 7. Статистика

```
Ученик → Фронтенд → Python → Go → Supabase → Go → Python → Фронтенд

1. Фронтенд запрашивает GET /api/v1/stats?user_id=...
2. Go:
   a. Считает total, correct, accuracy из attempts
   b. Группирует по topic
   c. Возвращает JSON
3. Фронтенд отображает графики/проценты
```

## Внутренняя архитектура Go

```
exam-trainer-backend/
├── cmd/
│   └── server/
│       └── main.go              # точка входа, запуск gin
├── internal/
│   ├── api/
│   │   ├── router.go            # маршруты (gin.Router)
│   │   ├── handlers.go          # обработчики запросов
│   │   └── middleware.go        # логирование, CORS
│   ├── models/
│   │   ├── task.go              # Task, Answer
│   │   ├── stats.go             # Stats
│   │   └── user.go              # User
│   ├── storage/
│   │   ├── supabase.go          # клиент Supabase (HTTP)
│   │   ├── tasks.go             # запросы к tasks
│   │   ├── attempts.go          # запросы к attempts
│   │   └── users.go             # запросы к users
│   ├── checker/
│   │   └── checker.go           # логика проверки ответов
│   └── adaptive/
│       └── adaptive.go          # адаптивная логика
├── migrations/
│   └── 001_init.sql             # создание таблиц
├── go.mod
└── README.md
```

### Ключевые файлы

| Файл | Что делает |
|------|------------|
| `main.go` | Запуск сервера, конфигурация |
| `router.go` | Маршруты: `/api/v1/tasks`, `/api/v1/check`, `/api/v1/stats` |
| `handlers.go` | Обработка HTTP-запросов, вызов storage/checker |
| `checker.go` | Сравнение ответов (choice, number, string, multi) |
| `adaptive.go` | Подбор уровня сложности по статистике |
| `supabase.go` | HTTP-клиент для Supabase REST API |

## API-контракты

### Go → Supabase

| Запрос | Метод | Путь Supabase |
|--------|-------|---------------|
| Задания | `GET` | `/rest/v1/tasks?subject=eq.{s}&topic=eq.{t}&difficulty=lte.{d}&limit={n}` |
| Проверка | `POST` | `/rest/v1/attempts` |
| Статистика | `GET` | `/rest/v1/attempts?user_id=eq.{id}&select=*` |
| Пользователь | `GET` | `/rest/v1/users?id=eq.{id}` |

### Python → Go

| Запрос | Метод | Путь Go |
|--------|-------|---------|
| Задания | `GET` | `http://localhost:8080/api/v1/tasks?...` |
| Проверка | `POST` | `http://localhost:8080/api/v1/check` |
| Статистика | `GET` | `http://localhost:8080/api/v1/stats?...` |
| Пользователь | `GET` | `http://localhost:8080/api/v1/user/{id}` |

### Python → AI

| Запрос | Провайдер | Модель |
|--------|-----------|--------|
| Подсказка | DeepSeek | `deepseek-chat` |
| Подсказка (фолбэк) | OpenAI | `gpt-4o-mini` |
| Генерация заданий | DeepSeek | `deepseek-chat` |

## Промты

### Подсказка (hint_system.md)

```
Ты — AI-ассистент для подготовки к ЕГЭ/ОГЭ.

Ученик решил задание неправильно. Твоя задача — дать направляющую подсказку, 
НО НЕ ОТВЕТ.

Задание: {task_text}
Правильный ответ: {correct_answer}
Ответ ученика: {user_answer}

Дай одну короткую подсказку (1-2 предложения), которая направит ученика 
к правильному ответу, но не раскроет его.
```

### Генерация заданий (generate_system.md)

```
Ты — генератор заданий для тренажёра ЕГЭ/ОГЭ.

Сгенерируй задание по предмету: {subject}
Тема: {topic}
Тип: {type}
Сложность: {difficulty}/5

Формат ответа (JSON):
{
  "text": "условие задания",
  "options": ["вариант1", "вариант2", "вариант3", "вариант4"],
  "answer": "правильный ответ",
  "explanation": "разбор"
}
```

## Безопасность

| Мера | Реализация |
|------|------------|
| Go не доступен снаружи | Только Python (5080) публичный |
| Лимит подсказок | `hints_used_today` в users, сброс каждые 24ч |
| Валидация ответов | Checker проверяет тип и формат |
| SQL-инъекции | Нет (Supabase REST API, не SQL) |
| XSS | Escape на фронте, Content-Security-Policy |

## Масштабирование

### Текущий лимит

- ~10 000 заданий
- ~100 одновременных пользователей

### Если растёт

| Проблема | Решение |
|----------|---------|
| Много запросов к Supabase | Кэширование (Redis) |
| Медленные AI-подсказки | Очередь задач (Celery) |
| Много пользователей | Вертикальный масштаб (больше CPU/RAM) |
| Нужна высокая доступность | Горизонтальный масштаб (контейнеры) |
