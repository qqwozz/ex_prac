
# Exam Trainer — Технологии

Вся техническая часть: стек, архитектура, БД, API, деплой. Без бизнес-логики.

---

## Архитектура

```
Пользователь
  │
  └──→ Python (5080) — фронт + AI
         │
         ├──→ Go (8080) — API + БД
         │       │
         │       └──→ Supabase (PostgreSQL)
         │
         └──→ DeepSeek → OpenAI (фолбэк)
```

| Сервис | Язык | Порт | Зона |
|--------|------|------|------|
| **Backend API** | Go 1.22+ / gin | 8080 | БД, проверки, API |
| **AI + Frontend** | Python / Flask | 5080 | Статика, AI-подсказки, генерация |
| **БД** | Supabase (PostgreSQL) | — | Хранение данных |

## Go

### Зона ответственности

- Бэкенд API
- Подключение к Supabase
- Проверка ответов
- Выдача заданий
- Статистика
- Адаптивная логика
- Работа с пользователями

### Стек

| Компонент | Решение |
|-----------|---------|
| Язык | Go 1.22+ |
| Фреймворк | gin |
| БД | Supabase (PostgreSQL) через HTTP API |
| Тесты | `testing` + `httptest` |

### API эндпоинты

| Метод | Путь | Что делает |
|-------|------|------------|
| `GET` | `/api/v1/tasks` | Получить задание (с фильтрами) |
| `POST` | `/api/v1/check` | Проверить ответ (Go или Python) |
| `GET` | `/api/v1/stats` | Статистика пользователя |
| `GET` | `/api/v1/user/{id}` | Данные пользователя (подписка) |
| `POST` | `/api/v1/tasks` | Добавить задание (форма/файл) |

### Таблицы в Supabase

| Таблица | Что хранит |
|---------|------------|
| `tasks` | Задания (условие, ответ, тип, сложность, тема) |
| `attempts` | Попытки учеников (кто, какое задание, ответ, результат) |
| `users` | Пользователи (подписка, реферал, лимит подсказок) |

### Форматы заданий (проверка)

| Тип | Кто проверяет | Как проверяет |
|-----|---------------|---------------|
| Выбор ответа (`choice`) | Go | Точное сравнение строки |
| Ввод числа (`number`) | Go | Сравнение с допуском (±0.01) |
| Ввод строки (`string`) | Go | Нормализация (нижний регистр, пробелы) + сравнение |
| Множественный выбор (`multi`) | Go | Совпадение множества |
| Написание кода (`code`) | Python | Запуск кода + сравнение вывода |
| Текстовый ответ (`text`) | Python | AI-анализ или эталонное сравнение |

**Логика:** Go проверяет простые типы мгновенно. Сложные типы (code, text) пересылаются POST-запросом на Python-сервер (`PYTHON_URL/ai/v1/check`).

### Адаптивное обучение

1. Считает % правильных ответов по каждому модулю
2. Сортирует модули по убыванию процента
3. **«Улучшить хорошее»**: берёт модули сверху → выдаёт задания сложнее
4. **«Подкачать плохое»**: берёт модули снизу → выдаёт задания проще

## Python

### Зона ответственности

- AI-подсказки
- Генерация заданий
- Проксирование запросов в Go

### Стек

| Компонент | Решение |
|-----------|---------|
| Язык | Python |
| Фреймворк | Flask |
| AI | DeepSeek (основной) → OpenAI (фолбэк) |

### AI-подсказки

| Параметр | Значение |
|----------|----------|
| Провайдеры | DeepSeek (основной) → OpenAI (фолбэк) |
| Лимит | 10 подсказок в день |
| Тип | Направляющая мысль, не ответ |
| Механика | По кнопке «Подсказка» |
| Хранение счётчика | Supabase: `users.hints_used_today` |

### Добавление заданий

| Способ | Когда |
|--------|-------|
| Веб-интерфейс (форма) | Добавить 1–5 заданий |
| Загрузка CSV/JSON | Массовая загрузка (100+) |

Формат файла: CSV или JSON, поля совпадают со схемой tasks.

### Генерация заданий

- ИИ создаёт задания для наполнения БД
- Формат: JSON, совпадает со схемой tasks
- Источник: `source = 'ai'`

### Telegram-бот для преподавателей

- Препод управляет заданиями через Telegram
- Бот позволяет: добавлять задания, смотреть статистику учеников
- Бот общается с Go API

## Frontend

### Стек

| Компонент | Решение |
|-----------|---------|
| Язык | HTML + CSS + Vanilla JS |
| Фреймворк | Нет (ноль зависимостей) |

### Страницы

| Страница | Что показывает |
|----------|----------------|
| Главная | Выбор предмета/режима |
| Задание | Текст задания, варианты, кнопка «Ответить» |
| Результат | Правильно/неправильно, объяснение, кнопка «Подсказка» |
| Статистика | % по модулям, общая статистика |
| Профиль | Реферальная ссылка, лимит подсказок |

### Как общается с бэкендом

| Путь | Куда идёт | Что делает |
|------|-----------|------------|
| `/api/v1/tasks` | Go (8080) | Получить задание |
| `/api/v1/check` | Go (8080) | Проверить ответ |
| `/api/v1/stats` | Go (8080) | Статистика |
| `/ai/v1/hint` | Python (5080) | AI-подсказка |
