
# Exam Trainer — Roadmap

Путь разработки от MVP до полноценного продукта.

---

## Общая философия

**MVP → итерации.** Сначала минимально рабочий продукт, потом наращиваем функционал. Каждая версия — рабочий продукт, который можно показать.

---

## MVP (v1) — «Рабочий тренажёр»

**Цель:** Ученик может решать задания и видеть результат.

### Go

| Задача | Статус |
|--------|--------|
| Настроить подключение к Supabase | 🔲 |
| Эндпоинт: GET /api/v1/tasks | 🔲 |
| Эндпоинт: POST /api/v1/check | 🔲 |
| Эндпоинт: GET /api/v1/stats | 🔲 |
| Проверка ответов (choice, number, string, multi) | 🔲 |
| Тесты на проверку ответов | 🔲 |

### Python

| Задача | Статус |
|--------|--------|
| Настроить Flask, отдачу статики | 🔲 |
| Проксирование /api/* → Go | 🔲 |
| AI-подсказки (DeepSeek + OpenAI фолбэк) | 🔲 |

### Frontend

| Задача | Статус |
|--------|--------|
| Список заданий | 🔲 |
| Ввод ответа + результат | 🔲 |
| Базовая статистика | 🔲 |

### Общее

| Задача | Статус |
|--------|--------|
| Наполнить ~100 заданий (математика + информатика) | 🔲 |
| Настроить Supabase (таблицы, тестовые данные) | 🔲 |

**Результат:** Ученик заходит, выбирает предмет, решает задания, видит правильный ответ и статистику.

**Длительность:** ~2–3 недели.

---

## v2 — «Адаптивное обучение»

**Цель:** Система подстраивается под уровень ученика.

### Go

| Задача | Статус |
|--------|--------|
| Адаптивная логика: подбор уровня сложности | 🔲 |
| Поддержка модулей (topic) в фильтрах | 🔲 |
| Эндпоинт: GET /api/v1/topics (список модулей) | 🔲 |
| Сортировка заданий по сложности | 🔲 |

### Python

| Задача | Статус |
|--------|--------|
| Telegram-бот: добавление заданий | 🔲 |
| Telegram-бот: просмотр статистики учеников | 🔲 |

### Frontend

| Задача | Статус |
|--------|--------|
| Экран выбора режима (улучшить/подтянуть) | 🔲 |
| Экран выбора модуля | 🔲 |
| Практика вразброс (смешанные модули) | 🔲 |
| Отображение прогресса по модулям | 🔲 |

**Результат:** Ученик выбирает режим, система подбирает задания. Препод управляет через Telegram.

**Длительность:** ~2–3 недели.

---

## v3 — «Монетизация и вовлечение»

**Цель:** Зарабатываем и удерживаем.

### Go

| Задача | Статус |
|--------|--------|
| Проверка подписки (basic/plus) | 🔲 |
| Таблица referrals, начисление бонусов | 🔲 |
| Таблица achievements (достижения) | 🔲 |
| Лидерборды (рейтинг учеников) | 🔲 |

### Python

| Задача | Статус |
|--------|--------|
| Расширение предметов (новые разделы) | 🔲 |

### Frontend

| Задача | Статус |
|--------|--------|
| Баннер «Оформите Плюс» для basic | 🔲 |
| Экран профиля с реферальной ссылкой | 🔲 |
| Игровой модуль UI (квесты, достижения) | 🔲 |
| Лидерборды UI | 🔲 |

**Результат:** Два тарифа, реферальная ссылка, игровые механики.

**Длительность:** ~3–4 недели.

---

## v4 — «Масштабирование»

**Цель:** Растим аудиторию и автоматизируем контент.

### Go

| Задача | Статус |
|--------|--------|
| JWT-авторизация (вместо user_id в URL) | 🔲 |
| Rate limiting | 🔲 |
| Оптимизация запросов (кэширование) | 🔲 |

### Python

| Задача | Статус |
|--------|--------|
| Генерация заданий ИИ с редактурой препода | 🔲 |
| Админка для преподов (веб, опционально) | 🔲 |

### Frontend

| Задача | Статус |
|--------|--------|
| Лидерборды, соревнования UI | 🔲 |
| Интеграция с основной школой (редирект) | 🔲 |

**Результат:** ИИ генерирует задания, ученики соревнуются, полная интеграция.

**Длительность:** ~4–6 недель.

## Дорожная карта (визуально)

```
MVP (2-3 недели)
  │
  ├──→ v2: Адаптив (2-3 недели)
  │       │
  │       ├──→ v3: Монетизация (3-4 недели)
  │       │       │
  │       │       └──→ v4: Масштабирование (4-6 недель)
  │       │
  │       └──→ Запуск для учеников школы
  │
  └──→ Первые пользователи
```

## Ключевые вехи

| Веха | Когда | Что получаем |
|------|-------|--------------|
| **Первый MVP** | Конец недели 1 | Рабочий тренажёр, можно показать |
| **Адаптив** | Конец недели 3 | Система подстраивается под ученика |
| **Монетизация** | Конец недели 6 | Два тарифа, рефералы |
| **Запуск** | Конец недели 10 | Полноценный продукт для школы |
| **Масштаб** | Конец недели 16 | Лидерборды, интеграция, генерация |

---

## Приоритеты

| Приоритет | Что | Почему |
|-----------|-----|--------|
| 🔴 Критично | MVP (задания + проверка + статистика) | Без этого нет продукта |
| 🟡 Важно | Адаптив | Ключевое отличие от конкурентов |
| 🟢 Желательно | Игровой модуль | Удержание, но не критично для старта |
| ⚪ Отложить | Лидерборды, генерация ИИ | Масштабирование, не для MVP |
